## 1. 修改目标

本次修改目标是限制 UFS 最大 Gear，避免设备进入更高 Gear 模式，用于安全 bring-up 或稳定性验证。

当前策略：

- Vendor bootloader 阶段：限制 UFS 最大 Gear 为 `G1`
- Kernel 阶段：限制 host capability 最大 HS Gear 为 `G1`
- 验证方式：通过 `/proc/ufs_debug` 查询实际 Gear 状态

---

## 4. 验证命令

设备开机后，进入 adb shell 查询 UFS 当前 Gear：

```bash
adb shell
cat /proc/ufs_debug | grep -i gear
```

---

## 5. 预期验证结果

如果限制生效，查询结果中应看到 RX / TX Gear 被限制到 `1`，类似：

```bash
[RX, TX]: gear=[1, 1]
```

如果仍然显示：

```bash
[RX, TX]: gear=[4, 4]
```

或：

```bash
[RX, TX]: gear=[5, 5]
```

则说明限制没有生效，需要继续确认：

1. 修改是否编译进当前 bootloader / kernel 镜像；
2. 设备实际启动的是否为最新烧录版本；
3. vendor 与 kernel 是否属于同一套代码版本；
4. `/proc/ufs_debug` 打印的是当前实际协商结果，还是初始化阶段最大能力值；
5. 是否还有其他位置覆盖了 `desired_pwr_mode->gear_rx / gear_tx`。

---

## 6. 本次修改总结

本次 UFS 限速主要修改两个位置：

| 模块 | 文件 | 修改内容 |
| --- | --- | --- |
| vendor / lk2 | `ufs_cfg.h` | `UFS_DEV_MAX_GEAR_RX/TX` 从 `4` 改为 `1` |
| kernel | `ufs-mediatek.c` | `host_cap.hs_rx_gear / hs_tx_gear` 从 `UFS_HS_G5` 改为 `UFS_HS_G1` |

最终通过以下命令确认实际 Gear：

```bash
adb shell "cat /proc/ufs_debug | grep -i gear"
```

目标结果：

```bash
gear=[1, 1]
```
